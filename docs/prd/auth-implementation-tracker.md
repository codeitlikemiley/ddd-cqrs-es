# Auth Implementation Tracker

## Status

implemented

## Goal

Track implementation progress for the fullstack auth solution across reusable
crates, Spin services, Leptos pages/forms, REST APIs, gRPC services, CLI
templates, storage, and verification.

## Non-Goals

- Do not replace the focused PRDs. Link to them instead.
- Do not use this file for design debate once a focused PRD owns the decision.
- Do not mark a task complete without evidence commands or file references.

## Success Criteria

- Every implementation slice has a status, owning PRD, expected files, and
  verification evidence.
- Another agent can resume from this tracker without needing chat history.
- `docs/docs.json` stays aligned with this tracker.

## Interfaces

Task statuses:

- `not_started`: no code or docs work has started.
- `in_progress`: active implementation or investigation is underway.
- `blocked`: unable to proceed without user input or external dependency.
- `operator_pending`: implementation and runbook are complete, but execution
  depends on external operator/provider state.
- `done`: implementation landed and verification evidence is recorded.
